# 设计说明

本项目刻意拆分为两部分：可移植 Skill 和一组小型确定性脚本。

Skill 负责给 Agent 提供教学框架，包括支持的考试类型、学习者基础水平、目标分数区间、八个助教角色、提示词模式、参考文档和模板。脚本负责稳定处理可度量的部分，包括档案校验、每日计划、练习记录、错误统计、能力画像更新、趋势分析、作文版本管理和作文评分参考。

## 公开安全发布

这个仓库适合发布到公开 GitHub，因为它发布的是角色边界和占位符，而不是私有提示词正文。公开文档可以说明每个助教做什么、如何路由任务，但不能复制、重写或重构原始八个助教提示词。

`full-local` 只是本地运行模式。它可以选择仓库之外的私有提示词资产，但这些资产必须留在本机私有环境中，不能进入共享文件。

## 确定性自动化

脚本故意避免概率式规划和随意评分，让学习者状态可以审计：

- `validate_profile.py` 拒绝不支持的考试类型、目标区间和缺失字段。
- `generate_daily_plan.py` 根据时间预算、目标考试、能力状态和错误证据生成受约束的每日计划。
- `record_practice.py` 使用明确的 `total_items` 和 `correct_items` 保存可复现练习记录。
- `summarize_errors.py` 按错误标签、模块和能力维度统计问题。
- `update_ability_profile.py` 把练习证据转化为能力画像变化。
- `score_writing_rubric.py` 给作文修改提供确定性评分参考，不代表官方评分。

## 计划模型

每日计划采用约束求解，而不是泛泛建议。计划器会考虑：

- 学习者考试类型和目标区间；
- 基础水平；
- 每日可用时间；
- 状态较弱的能力节点；
- 重复出现或影响较大的错误标签；
- 词汇、听力、阅读、翻译、写作之间的模块覆盖。

Agent 可以调整面向学习者的表达方式，但计划里的模块、分钟数、重点和原因应与脚本输出一致，除非学习者修改约束。

## 持续学习设计原则的适配范围

ExamLex 将 Nuwa Skill、book-to-skill、Cangjie Skill、video-downloader 和 Darwin Skill 的思路适配到英语考试策略库，但不宣称与这些项目功能完全等价。

| 上游原则 | ExamLex 中的落实方式 |
| --- | --- |
| Nuwa：多角度采集、三重验证、五层认知提取和诚实边界 | `prompts/cognitive.py`、来源溯源、RIA 边界字段和校验测试 |
| book-to-skill：不直接倾倒原文、按章节渐进加载、提取术语表、使用可执行表达 | 书籍提取器的章节/术语表产物、RIA 提示和具体性检查 |
| Cangjie：Adler 整体分析、并行方法提取、RIA-TV++、策略关联和压力测试 | `prompts/ria.py`、`related_strategies`、RIA Schema 校验和效果评估 |
| video-downloader：稳定的视频/转写产物契约和明确的媒体工具边界 | `yt-dlp` 下载/元数据、`ffmpeg` 合并媒体流及转换音频、本地 `whisper` 或 SiliconFlow 语音识别 |
| Darwin：验证门控、可度量评分、棘轮保留、早停和可审计证据 | 确定性结构评分、Agent 效果测试、阈值门控提交、不可变修订、审批哈希和分数历史 |

Python 脚本强制保证的是确定性校验、哈希、不可变修订引用、审批阈值和棘轮行为。多 Agent 调研、并行提取、压力测试执行和独立评委仍由 Agent 工作流编排；当前 CLI 会记录并校验其产物，但不会自行启动或证明每轮都使用了两名全新的独立评委。视频 URL 当前只支持 B 站和 YouTube，不宣称覆盖上游 video-downloader 的全部平台。
