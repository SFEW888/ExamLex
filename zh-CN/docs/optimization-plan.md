# 优化计划

本文档记录已经完成的优化工作和剩余发布优先级。README 只描述已经存在并有测试覆盖的命令。

## 已在本仓库实现

- Codex、Claude Code、Codex App 和 Cursor 的 Agent Skill 调用方式。
- 终端快捷封装：`bin/examlex` 与 `bin/examlex.ps1`。
- 核心学习闭环：档案校验、受约束的每日计划、练习记录、错误归因、汇总、能力更新、趋势、作文版本和作文评分。
- 策略库支持 JSON 与事务型 SQLite 存储、导入导出、重复候选提示和容量预警；重要策略与历史版本绝不自动删除。
- 本地数据备份与恢复。
- 使用随包参考数据的词汇量估算，以及四级 3,331 词、六级 3,650 词、考研英语 1,014 词的实际计数扩展池。
- 限时练习元数据与间隔复习调度。
- HTML 学习进度报告。
- 范文、写作参考和随 wheel 打包的运行资源。
- 陈旧会话安全归档、可再生成产物硬上限、每 30 分钟后台容量任务与运维就绪检查。
- 用于确定性测试和诊断的离线运维检查。
- SiliconFlow multipart 流式上传、备份源文件单次哈希读取和线性章节偏移检测。
- 目录剪枝与单次仓库文件索引、Ruff、分支覆盖率门禁和非重复 CI 构建。
- 以 `skills/examlex/` 为唯一实现和资源包，`examlex/` 仅保留轻量兼容入口，避免脚本和资源重复占用仓库容量。

## 下一阶段优先级

1. 持续提高当前低覆盖率 CLI、可视化和词汇模块的测试覆盖。
2. 深化 TEM-4、TEM-8 工作流示例与回归覆盖。
3. 在代表性设备上对大型本地备份和云端音频上传进行基准测试。
4. 持续执行公开发布检查表、wheel 隔离安装和版本兼容验证。
5. 扩充适用于不同基础学习者的本地示例工作流。
